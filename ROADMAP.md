# Evergreen — Роадмап: «Цифровой операционный директор»

> Большая идея: Evergreen перестаёт быть просто справочником для сотрудников и
> становится **цифровым опер-директором для собственника сети**. Не «витаминка»,
> а «обезболивающее» — заменяет работу живого опер-дира (300–600к/мес). Это ровно
> то, что обещает лендинг («цифровой двойник руководителя»).

---

## ✅ Уже сделано (готово к показу клиенту)

- Чат по стандартам со строгим RAG (отвечает по делу, на чужое честно отказывает).
- Живая модель Gemini (эмбеддинги + генерация) — работает.
- Фронт отдаётся из бэка, всё на одном порту (:8000).
- Микро-обучение: карточки со свайпом, авто-сборка курса, назначение, прогресс.
- Тесты к урокам + сокрытие правильных ответов (проверка на сервере).
- Поиск «пробелов»: когда бот не может ответить сотруднику → пишется в gaps.

## 🔨 ДЕЛАЕМ СЕЙЧАС: Командный центр CEO на отзывах 2GIS

Первый билд цифрового опер-дира. Командный центр = экран собственника;
отзывы 2GIS = топливо для него.

**Что делает собственник:** вставляет ссылку на точку в 2GIS → жмёт «Обновить
отзывы» → открывает «Командный центр».

**Что видит на экране:**
- Пульс точки: сколько отзывов за период, средняя оценка, сколько негативных.
- Главные боли: жалобы, сгруппированные и привязанные к стандартам
  (напр. «Чистота — 3 жалобы → нарушается Регламент уборки п.1.3»).
- Что делать: короткие рекомендации AI по каждой боли.
- Лента отзывов с AI-метками (тема + тональность).

**Порядок работы (по шагам):**
1. [ ] Парсер 2GIS (по ссылке → список отзывов), спрятан за «адаптер источника».
2. [ ] AI-анализ отзыва: тема + тональность + связь со стандартом + рекомендация
       (через `brain.complete`, как генерация теста; строгий JSON-контракт).
3. [ ] Хранение отзывов и разбора в БД (новые таблицы).
4. [ ] API для командного центра.
5. [ ] Экран «Командный центр» (только владелец).

**Источник отзывов:** пока ТОЛЬКО 2GIS, через парсинг.

**Два честных риска:**
- Парсинг 2GIS хрупкий и в серой зоне (внутренний API может меняться, формально
  против их правил). Для первого клиента/демо — ок. Прячем за адаптер, чтобы
  потом легко уйти на официальный API или ручную загрузку.
- Анализ живой моделью (Gemini) запускает только владелец ключа. Я строю + проверяю
  офлайн на фейковых отзывах и тестах; пользователь подтверждает качество вживую.

## ⏭️ ЧТО ДАЛЬШЕ — вся работа опер-дира, по очереди

Цель: оцифровать всё, что делает живой операционный директор.

1. **AI-соавтор стандартов + поиск пробелов.** CEO в чате создаёт/правит стандарты;
   AI находит дыры (в т.ч. из gaps — вопросов, на которые бот не ответил) и
   предлагает дополнения. *(Частично есть: gaps уже копятся.)*
2. **Сеть точек: мониторинг и сравнение.** Несколько филиалов; видно, какая точка
   проседает, сравнение между ними.
3. **Контроль качества: чек-листы и аудиты.** Регулярные проверки точек по стандартам
   (тайный покупатель, чек-листы смены).
4. **Авто-источники отзывов.** Официальный 2GIS API, Google, Yandex — вместо парсинга.
5. **Сводки и тревоги собственнику.** Еженедельный дайджест «как дела в сети» +
   пуш-алерты на резкий негатив.
6. **Отслеживание исправлений.** Назначить фикс управляющему точки, проследить, что
   проблема реально закрыта (петля: нашли → поручили → проверили).
7. **Геймификация обучения** (XP, серии, значки, лидерборд) — backlog, отложено.

### Чек-лист «что делает опер-дир» (чтобы ничего не забыть оцифровать)
- [x] Обучает и онбордит персонал
- [~] Держит стандарты/регламенты актуальными (соавтор — в планах)
- [~] Находит пробелы и риски в стандартах (gaps есть, разбор — в планах)
- [ ] Слушает клиентов (отзывы, жалобы) ← **делаем сейчас (2GIS)**
- [ ] Контролирует качество по точкам (аудиты, чек-листы)
- [ ] Сравнивает филиалы между собой
- [ ] Докладывает собственнику: где болит, что чинить
- [ ] Следит, что исправления реально сделаны

## ⚠️ Постоянные ограничения (для любого, кто продолжает)
- НЕ трогать `composer/` — это движок, продукт ходит в него только через `product/brain.py`.
- Бренд везде «Evergreen». Никогда не светить «Composer AI» / внутренности харнесса.
- Живые вызовы LLM запускает только владелец ключа (ключ в его .env, не в репо, не в образе).
- Коммитить только по явной просьбе пользователя.
