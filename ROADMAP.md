# Evergreen — Роадмап: «Цифровой операционный директор»

> Большая идея: Evergreen перестаёт быть просто справочником для сотрудников и
> становится **цифровым опер-директором для собственника сети**. Не «витаминка»,
> а «обезболивающее» — заменяет работу живого опер-дира (300–600к/мес). Это ровно
> то, что обещает лендинг («цифровой двойник руководителя»).

## ⏭️ ЧТО ДАЛЬШЕ — вся работа опер-дира, по очереди

Цель: оцифровать всё, что делает живой операционный директор.

1. **AI-соавтор стандартов + поиск пробелов.** CEO в чате создаёт/правит стандарты;
   AI находит дыры (в т.ч. из gaps — вопросов, на которые бот не ответил) и
   предлагает дополнения. *(Частично есть: gaps уже копятся.)*
2. ~~**Сеть точек: мониторинг и сравнение.**~~ ✅ Сделано: несколько филиалов,
   видно, какая точка проседает и какая лучшая, сравнение по метрикам.
3. **Контроль качества: чек-листы и аудиты.** Регулярные проверки точек по стандартам
   (тайный покупатель, чек-листы смены).
4. **Авто-источники отзывов.** Официальный 2GIS API, Google, Yandex — вместо парсинга.
5. ~~**Сводки и тревоги собственнику.**~~ ✅ Сделано: экран «Сводка» — пульс сети,
   детерминированные тревоги по проседающим точкам (низкий рейтинг, много жалоб,
   негатив преобладает, повторяющаяся боль) + короткий брифинг (модель поверх уже
   посчитанных фактов, офлайн-фолбэк). Пуш-алерты (e-mail/Telegram) — отдельным шагом.
6. **Отслеживание исправлений.** Назначить фикс управляющему точки, проследить, что
   проблема реально закрыта (петля: нашли → поручили → проверили).
7. **Геймификация обучения** (XP, серии, значки, лидерборд) — backlog, отложено.

## 🧠 Ядро движка (Composer) — вложения, которые масштабируют Evergreen

> Контекст: движок умеет многошаговых агентов с инструментами и оркестрацией,
> но Evergreen пока ходит в него только через одиночный `brain.complete`.
> Цель — постепенно «включить» возможности ядра под продукт.

- [x] **Строгий вывод по схеме (`complete_json`).** В ядре —
      `composer/engine/structured.py` (извлечение JSON из «грязного» ответа +
      переспрос модели при битом формате). В продукте — `brain.complete_json`.
      Переведены фичи: генерация теста (M3) и разбор отзывов (M4). Покрыто
      тестами. Это поднимает надёжность всех фич и готовит почву под агента.
- [x] **Агент-опер-дир с инструментами (#1, главный рычаг).** Сделано:
      `brain.run_tools_agent` (шов: продукт отдаёт движку СВОИ callable-инструменты,
      движок крутит многошаговый цикл), модуль `product/modules/advisor.py` с
      read-only инструментами, скоупленными на компанию (search_standards,
      network_overview, point_problems, list_gaps), API `POST /api/advisor`
      (require_manager) и экран «Опер-дир» (чат). Покрыто офлайн-тестами
      (скриптованный провайдер). Живой прогон на Gemini подтверждает владелец ключа.
- [ ] **Учёт расхода и лимиты (#3).** Подсчёт токенов/стоимости на компанию +
      предохранитель бюджета — для экономики SaaS.

## ⚠️ Постоянные ограничения (для любого, кто продолжает)
- НЕ трогать `composer/` — это движок, продукт ходит в него только через `product/brain.py`.
- Бренд везде «Evergreen». Никогда не светить «Composer AI» / внутренности харнесса.
- Живые вызовы LLM запускает только владелец ключа (ключ в его .env, не в репо, не в образе).
- Коммитить только по явной просьбе пользователя.
