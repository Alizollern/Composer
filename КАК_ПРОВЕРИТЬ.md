# Как проверить Evergreen самому (флоу для руководителя)

Этот файл — простыми словами, без чтения кода. Две части:
**А** — быстрая проверка за 10 секунд (без ключей и без интернета).
**Б** — полноценный запуск с реальным ИИ и проверка глазами в браузере.
И **В** — как читать «журнал мыслей» ИИ, чтобы его контролировать.

---

## А. Быстрая проверка без ключа (10 секунд)

Я уже прогнал её за вас — всё зелёное. Вы можете повторить одной командой
в терминале из папки проекта:

```
python3 scripts/test_flow.py
```

Что эта команда делает (играет за целую компанию «Bronx Fitness»):

1. Поднимает сервис и проверяет, что он живой.
2. **Регистрирует компанию и владельца** — как форма «Создать компанию».
3. **Заводит сотрудника** (администратора зала).
4. **Загружает 2 стандарта** в Базу знаний и публикует их.
5. Сотрудник **входит** и видит опубликованные стандарты.
6. Сотрудник **спрашивает бота то, что ЕСТЬ в базе** → бот отвечает и
   ссылается на документ-источник.
7. Сотрудник **спрашивает то, чего НЕТ** → бот **честно отказывается** и не
   выдумывает (это наш главный принцип — строгий ответ только по стандартам).
8. Владелец открывает **«Карту пробелов»** — там лежит вопрос без ответа.
9. По стандарту **генерируется тест**, сотрудник его проходит.
10. Собирается **трек онбординга**, сотрудник зачисляется и проходит шаг.
11. Владелец видит **прогресс команды**.

Если внизу написано «✅ ВСЁ ПРОШЛО» — бэкенд работает целиком.

> Важно: тут ИИ подменён «заглушкой», чтобы не тратить ключ и не зависеть от
> интернета. Проверяется вся **логика и проводка** (доступы, гейты, ссылки на
> источники, тесты, треки). Качество самих формулировок ответа проверяется в
> части Б, где работает настоящая модель.

## Б. Полноценный запуск с реальным ИИ (проверка в браузере)

Здесь нужен ваш ключ (он есть только у вас в терминале — это правильно).

1. Один раз скопируйте файл настроек и впишите ключ:
   ```
   cp .env.example .env
   ```
   Откройте `.env` и впишите:
   - `GEMINI_API_KEY=` — ваш бесплатный ключ Google AI Studio (для ответов и
     для эмбеддингов; самый простой путь);
   - либо `ANTHROPIC_API_KEY=` и поменяйте `COMPOSER_PROVIDER=claude` — если
     хотите отвечать через Claude.

2. Поднимите всё одной командой (Postgres + бэкенд + собранный фронт):
   ```
   docker compose up --build
   ```

3. Откройте в браузере **http://localhost:8000** и пройдите вручную:
   - **Зарегистрировать компанию** → придумайте название и slug (англ. буквы),
     почту и пароль владельца.
   - Зайдите в **Базу знаний** → **Загрузить документ** → закиньте реальный
     стандарт (например, регламент администратора).
   - Откройте **Ассистента** и задайте вопрос по этому документу → бот должен
     ответить и показать **источник**.
   - Задайте вопрос «не по теме» → бот должен **честно отказать**.
   - Этот отказ появится у владельца в списке пробелов.

Если на каждом шаге поведение совпало — система работает вживую.

## В. Как контролировать ИИ — журнал «мыслей» агента

Каждый раз, когда бот что-то делает, он пишет одну понятную строку в журнал:
что нашёл в базе, на чём решил ответить и что именно спросил у модели.

Посмотреть журнал по-человечески:

```
python3 scripts/agent_log.py            # весь журнал
python3 scripts/agent_log.py -n 20      # последние 20 событий
python3 scripts/agent_log.py -f         # следить вживую, пока пользуются ботом
```

Пример того, что вы увидите:

```
🔎 ПОИСК по базе → лучшее совпадение 0.41 (порог 0.25)
        кандидаты: Стандарт администратора=0.41, Стандарт уборки=0.0
🤖 ВЫЗОВ МОДЕЛИ «chat_strict_rag» → ответ: По стандарту... Источник: «...»
✅ ОТВЕТИЛ (гейт «generation»)  источники: Стандарт администратора
...
🔎 ПОИСК по базе → лучшее совпадение 0.0 (порог 0.25)
🚫 ОТКАЗАЛ (гейт «retrieval»)
```

Так видно **каждое решение бота**: почему ответил, на основании какого
документа, и почему отказал. Это и есть контроль ИИ.

- Журнал лежит в `var/logs/agent.jsonl` (в репозиторий не попадает).
- При запуске через `docker compose` журнал виден на вашем компьютере в той же
  папке `var/logs/` — можно открыть `python3 scripts/agent_log.py -f` в соседнем
  окне терминала и наблюдать за ботом в реальном времени.
- Выключить журнал: переменная окружения `EVERGREEN_AGENT_LOG_ENABLED=0`.
